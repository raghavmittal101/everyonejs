Implementation

We will implement a small prototype which will be a JavaScript API using which a developer can embed different tools in the frontend code to generate desired UX for user with disabilities.

1 Requirements

1.1 For Voice I/O (for type C users, type F users)

  • API for text to speech synthsis
  • API for voice recognition

1.2 For type D users

  • Multicolor options on CSS files

1.3 All output sounds to visual test display (For type A users)

  • API for speech(capture sounds generated by software) to text conversion.

1.4 For performing user Accessibility test

  • create a small demo like test for starting a an app in accessibility mode, which will ask user to give voice input or choose an option. This will ensure the type of user.

2 Technical specs

2.1 Prototype development sprints classified according to I/O type

I/O type milestone

2.2 Available resources

  • API for text to speech - google speech synthesis API
  • API for speech to text - annyang

2.3 Implementation

2.3.1 Page Reader

  1. API
    class name functionality
    speak to identify an element as a voice output
    speak-alt to identify alt attribute of an element as voice output.
    seq-n to mark the sequence number n of an element. It is the order in which it should be narrated among multiple narratable elements.
